Một issuer, nhiều RP Tự viết OIDC (Go) SDS mã hóa PII Postgres-first DDD / Clean Architecture Kafka · Redis · K8s

01 Tổng quan

Ba trụ cột, theo mô hình tập trung (gần UAE PASS), khác hướng SSI/blockchain của NDA Key.

🔐 IdP / SSO

OAuth 2.0 / OpenID Connect tự viết. Một issuer, một pool user, nhiều RP/client. Đăng nhập một lần xuyên mọi ứng dụng.

🪪 eKYC + IAL

Xác minh người thật qua adapter nhà cung cấp (FPT.AI…). Nâng mức định danh IAL1 (email/SĐT) → IAL2 (đã eKYC).

🛡️ Sensitive Data Service

Service cô lập, mã hóa toàn bộ PII/KYC bằng mô hình khóa phân cấp có version. Điểm khác biệt mạnh nhất.

03 Luồng đăng nhập OIDC

Authorization Code + PKCE, một issuer. Token JWT ES256 validate offline qua JWKS.

sequenceDiagram autonumber participant U as Người dùng participant RP as Relying Party participant IC as identity-core (IdP) RP->>IC: /authorize (client_id, PKCE, state, nonce) IC->>U: Trang login + consent U->>IC: Passkey / mật khẩu + MFA IC-->>RP: authorization code RP->>IC: /token (code + code_verifier) IC-->>RP: ID/Access JWT (ES256, claim ial) + refresh Note over RP,IC: RP validate iss / aud / chữ ký qua JWKS (offline)

04 Luồng eKYC → nâng IAL

PII không bao giờ ở identity-core dạng plaintext — chỉ giữ kyc_id + ial.

sequenceDiagram autonumber participant U as Người dùng participant IC as identity-core participant P as eKYC provider participant S as SDS U->>IC: Upload giấy tờ + selfie IC->>P: Xác minh (OCR + face + liveness) P-->>IC: Kết quả khớp IC->>S: Lưu PII (mã hóa AES-256-GCM) S-->>IC: kyc_id IC->>IC: Nâng IAL1 → IAL2

08 Roadmap

MVP — build mỏng
  • OIDC code+PKCE (một issuer), JWKS + xoay khóa
  • Login + Passkey/FIDO + MFA
  • Token JWT ES256, refresh rotation + reuse-detection
  • eKYC qua provider → IAL1 / IAL2 (gọi SDS)
  • SDS: MASTER_KEY→DATA_KEY(version)→HKDF, AES-256-GCM, searchable hash, RBAC theo kyc_id
  • Đăng ký & quản trị client/RP, user; audit (Postgres)
  • Stateless + Redis + rate limit + object storage
Phase 2 — đích
  • Tích hợp VNeID (đang cần quyết MVP vs roadmap)
  • Chữ ký số pháp lý (Luật GDĐT 2023)
  • Kho tài liệu / ví credential
  • Verifiable Credentials + ZKP (hướng lai SSI)
  • SDK cho bên tích hợp · social login
  • Sharding thật · đa-DC/GEO · autoscale · MongoDB cho audit
  • Multi-tenant (nếu bán IdP cho tổ chức ngoài)
