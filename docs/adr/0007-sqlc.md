# ADR-0007: sqlc + pgxpool cho truy cập DB

**Tác giả:** Thành Lê Phước

- **Trạng thái:** Accepted
- **Ngày:** 2026-06-07

## Bối cảnh
Cần lớp truy cập Postgres. Lựa chọn: sqlc (sinh code type-safe từ SQL thuần), Bun (ORM nhà), GORM (ORM), pgx thuần. House quen Bun.

## Quyết định
Dùng **sqlc** (sinh code Go type-safe từ SQL thuần) + **pgxpool** (driver/pool) + **goose** (migration).

## Hệ quả
- **Tích cực:** SQL tường minh, type-safe, kiểm soát truy vấn — quan trọng với truy vấn nhạy cảm bảo mật + RLS/partition của Postgres; tránh "ẩn SQL" và bẫy N+1 của ORM.
- **Tiêu cực:** lệch cơ-tay team (house quen Bun); cần bước codegen.

## Phương án đã cân nhắc
- **Bun (house):** khớp team hơn, nhưng ẩn SQL — đánh đổi kiểm soát; có thể dùng nếu team ưu tiên cơ-tay (điểm này để ngỏ, dễ đổi).
- **GORM:** ORM nặng, ẩn SQL, khó kiểm soát truy vấn nhạy cảm.
- **pgx thuần:** kiểm soát tối đa nhưng nhiều boilerplate scan row.
